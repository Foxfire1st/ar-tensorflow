## What This Area Is

MLIR transform and legalization code for TensorFlow-to-XLA bridge paths. This
route contains TableGen rewrite definitions, generated pass declarations, C++
pass implementations, target legality helpers, tests, and fallback bridges to
classic TF2XLA lowering.

## Hot Path Summary

Use this route for TensorFlow dialect to XLA/HLO legalization. Strong anchors are `legalize_tf_patterns.td`, `TF_CheckNumericsOp`, `legalization_op_config.cc`, `legalize_tf.cc`, `generated_legalize_tf.inc`, `xla_legalize_tf.cc`, and `tf2xla_rewriter.cc`.

## Structures Found Here

The `BUILD` file generates `generated_legalize_tf.inc` from
`legalize_tf_patterns.td`, generates pass headers from `.td` pass definitions,
builds `legalize_tf`, builds `xla_legalize_tf`, and defines tests for target
legality and verification. The route depends on TensorFlow dialect definitions,
StableHLO/CHLO ops, MLIR core dialects, and XLA HLO translation utilities.

## Operating Model

Declarative rewrite patterns handle direct TensorFlow dialect rewrites into
StableHLO/CHLO or related HLO dialect operations. C++ pass code contributes
helper functions, target legality checks, and bridge-specific fallback. The
Bazel `gentbl_cc_library` target turns the `.td` pattern file into an include
that the C++ pass compiles.

## Main Flows

### Generated Pattern Legalization

1. `legalize_tf_patterns.td` defines TableGen `Pat` rewrites and native helper
   calls.
2. The `legalize_tf_patterns_inc_gen` target runs `mlir-tblgen -gen-rewriters`.
3. `legalize_tf.cc` compiles the generated include and helper code into the
   `legalize_tf` library.

### Bridge Legalization With TF2XLA Fallback

1. C++ passes attempt direct MLIR legalization where a pattern exists.
2. Unsupported or special ops can route through TF2XLA registry-based rewriting.
3. Verification tests assert target legality and expected rewrites.

## Local Invariants And Traps

- Changes to `legalize_tf_patterns.td` affect generated C++ through
  `generated_legalize_tf.inc`.
- NativeCodeCall helpers named in the `.td` file must exist in the C++ helper
  scope compiled with the generated patterns.
- Direct rewrite patterns and TF2XLA fallback are separate mechanisms; do not
  assume a missing direct pattern means an op is unsupported.
- Pattern comments sometimes document known semantic gaps. Preserve them when
  touching adjacent patterns.

## Purpose

Auto-generated FlatBuffer header defining the TFLite model schema — the wire format for `.tflite` files. Generated from `tensorflow/lite/schema/schema.fbs`. Contains C++ structs for `Model`, `SubGraph`, `Operator`, `Tensor`, `Buffer`, `OperatorCode`, and all metadata/quantization types. `FlatBufferModel` uses these generated types to parse and validate `.tflite` files.

This file is generated by the `flatc` FlatBuffer compiler and should not be hand-edited.

## Code Commentary

### Logic

**Generated types (key ones):**
- **`Model`** — top-level: `version`, `operator_codes`, `subgraphs`, `buffers`, `metadata`, `signature_defs`.
- **`SubGraph`** — tensors, operators, inputs/outputs indices.
- **`Operator`** — `opcode_index`, `inputs`, `outputs`, `builtin_options`, `custom_options`.
- **`Tensor`** — `shape`, `type`, `buffer`, `quantization`, `is_variable`.
- **`Buffer`** — raw data bytes (weights).
- **`OperatorCode`** — `builtin_code` or `custom_code` (string).
- **`QuantizationParameters`** — scale, zero_point, min/max values.

Generated by running `flatc` on `schema.fbs`.

### Conventions

- **FlatBuffers format** — uses FlatBuffers for zero-copy deserialization, unlike the rest of TF which uses protobuf.
- **Auto-generated** — modifications go in `schema.fbs`, not here.

### Invariants And Boundaries

- Auto-generated by flatc from schema.fbs. Do not edit manually.
- Schema version is embedded in every model file; runtime checks compatibility.
- All FlatBuffer types (Model, SubGraph, Operator, Tensor) are defined here.

- **Do not edit** — changes must be made in the `.fbs` schema file and regenerated.
- **Version compatibility** — the schema version is checked against `TFLITE_SCHEMA_VERSION` at load time.
- **FlatBuffer wire format** — provides backwards-compatible field evolution via table-based layout.
- **Do not edit this file** — Changes are overwritten on the next flatc regeneration. Modifications to the schema must go in schema.fbs.
- **Version mismatch** — A schema_generated.h compiled against schema v3.0.0 may fail to load models with v3.1.0 features. Runtime compatibility is best-effort.
- **The FlatBuffer Verifier** — Model::Verify() must be called before access. Unverified models may trigger OOB reads. Some older versions skip verification for performance (dangerous).
